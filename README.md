# KIDA_ADSIM: 한국군 방공체계 C2 아키텍처 비교 시뮬레이션

## 1. 프로젝트 목표

한국군 방공체계의 두 가지 지휘통제(C2) 아키텍처 — **선형 C2**와 **Kill Web** — 의
방공 효과를 정량적으로 비교·분석하기 위한 M&S(Modeling & Simulation) 프레임워크.

### 핵심 질문
> "분산형 Kill Web 아키텍처가 기존 계층형 선형 C2 대비
> 탐지-교전 시간, 누출률, 교전 성공률, 체계 회복력에서 실질적 우위를 보이는가?"

### 현재 버전: v0.6b (완료)
- Monte Carlo 4,200회 배치 실험 + 통계 분석 프레임워크 완성
- 13개 테스트 파일, 136개 테스트 전부 PASS
- 14개 GIF 전술 애니메이션 + 21개 비교 스냅샷 생성

### 연구 배경
- 기존 한국군 방공체계는 **계층형 선형 C2** 구조 (감시→방공통제소→대대→발사대)
- 북한의 SRBM(KN-23), 순항미사일, UAS 군집 등 **다중·동시 위협** 증가
- JADC2/Kill Web 개념의 한국군 적용 가능성 검토 필요

## 2. 비교 대상 아키텍처

### 2.1 선형 C2 (Hierarchical Linear)
```
[조기경보레이더] → [방공통제소(MCRC)] → [대대전술작전반(TOC)] → [발사대]
```
- 순차적 킬체인: 센서→C2→C2→사수 (단일 경로)
- MCRC가 단일장애점(SPOF) — 피격 시 체계 마비
- 교전 승인에 15~120초 소요
- 평균 Sensor-to-Shooter(S2S): **~352초**

### 2.2 Kill Web (Distributed Mesh)
```
[모든 센서] ↔ [모든 C2] ↔ [모든 사수]  (완전 메시)
```
- 병렬 킬체인: 다중 경로 동시 처리
- 분산 구조로 단일장애점 없음 — 노드 손실 시 자동 우회
- 자동화된 교전 승인 (2~5초)
- 평균 S2S: **~5초** (선형 대비 70배 향상)

## 3. 시뮬레이션 구조

### 3.1 에이전트 기반 모델 (Agent-Based Model)

| 에이전트 | 종류 | 주요 속성 |
|----------|------|-----------|
| **센서** | EWR, PAT 레이더, MSAM MFR, SHORAD | 탐지거리, 추적용량, RCS 보정 |
| **C2 노드** | MCRC, 대대 TOC, EOC | 처리용량, 승인지연, 공통작전상황도 |
| **사수** | PAC-3 MSE, 천궁-II, 비호(gun/missile 이중모드), KF-16 | 사거리, 최대교전고도, Pk 테이블, 탄약 |
| **위협** | SRBM, 순항미사일, 항공기, UAS | 속도, 고도, RCS, 비행 프로파일 |

### 3.2 기술 스택

| 프레임워크 | 역할 |
|-----------|------|
| **Mesa 3.0** | 에이전트 기반 모델링 (ABM) |
| **SimPy 4.1** | 이산사건 시뮬레이션 (킬체인 지연·큐잉) |
| **NetworkX 3.0** | C2 토폴로지 구축 (선형/메시) |
| numpy, pandas, scipy | 통계 분석 |
| matplotlib, seaborn | 시각화 |

### 3.3 시뮬레이션 흐름 (매 5초 스텝)
```
1. 위협 활성화 및 이동 (비행 프로파일 기반 고도·속도 갱신)
2. 센서 탐지 (RCS·거리·재밍 반영)
3. 킬체인 프로세스 시작 (SimPy 지연 모델링)
4. 교전 승인 대기 (아키텍처별 차등)
5. 사수 할당 및 교전 (Pk 기반 베르누이 시행)
6. 결과 기록 (격추/누출/재교전)
```

## 4. 위협 모델

### 4.1 위협 유형

| 위협 | 모델 | 속도 | 고도 범위 | RCS | 특성 |
|------|------|------|-----------|-----|------|
| **SRBM** | KN-23형 | Mach 6 | 0→50→0 km | 0.5 m² | 탄도궤적 3단계 (부스트/중간/종말) |
| **순항미사일** | 금성-3형 | Mach 0.8 | 0→0.5→0.03 km | 0.05 m² | 해면밀착 순항 + 종말 기동 |
| **항공기** | 고정익 | Mach 1.0 | 10→1→0.5 km | 5.0 m² | 고고도 접근→저고도 침투 |
| **UAS** | 군집 드론 | 180 km/h | 0.3→0.05 km | 0.005 m² | 저고도 은밀 접근 |

### 4.2 비행 프로파일 (단계별 동적 모델)
각 위협은 비행 단계(phase)에 따라 고도·속도·기동 여부가 동적으로 변화:
- **SRBM**: 부스트(0→50km, 가속) → 중간비행(50→30km) → 종말(30→0km, 풀업 기동)
- **순항미사일**: 부스트(0→0.5km) → 해면밀착 순항(30m) → 종말 기동
- **항공기**: 고고도 순항(10km) → 저고도 침투(1km) → 공격 진입
- **UAS**: 저고도 접근(300m) → 종말 강하(50m)

## 8. 실행 방법

```bash
# 의존성 설치
pip install -r requirements.txt

# 빠른 실행 (시나리오 1, 두 아키텍처 비교)
python -c "
from modules.model import AirDefenseModel
for arch in ['linear', 'killweb']:
    m = AirDefenseModel(architecture=arch, scenario='scenario_1_saturation', seed=42)
    r = m.run_full()
    s2s = r['metrics']['sensor_to_shooter_time']['mean']
    leaker = r['metrics']['leaker_rate']
    success = r['metrics']['engagement_success_rate']
    print(f'{arch}: leaker={leaker:.1f}%, s2s={s2s:.1f}s, success={success:.1f}%')
"

# 전체 테스트 (136개)
python -m pytest tests/ -v

# Monte Carlo 파일럿 실행 (시나리오 1, 10회)
python -c "
from modules.batch import BatchRunner
br = BatchRunner(scenarios=['scenario_1_saturation'], n_runs=10)
results = br.run_pilot()
print(results)
"
```

## 9. 핵심 아키텍처 비교 결과 (v0.6a, seed=42)

| 항목 | 선형 C2 | Kill Web |
|------|---------|----------|
| S2S 시간 | ~352s | ~5s (70배 빠름) |
| 누출률 (S1) | 37.8% | **35.6%** |
| 토폴로지 | 계층 체인 (MCRC SPOF) | 완전 메시 (분산) |
| COP 품질 | 위협 항적만 | **위협 + 아군 상태 + 교전 계획** |
| 센서 융합 | 단일 센서 (σ=0.5km) | **√N 오차 감소 (min 0.1km)** |
| 사수 선택 | 유형 우선 고정 | **COP 기반 최적 점수** |
| 교전 정책 | 고정 다중 교전 | **적응형 (탄약 기반 자동 전환)** |
| 통신 열화 | 전역 균일 | **링크별 차등 + 메시 다중경로 완화** |

> S2 복합위협: KillWeb 13.2% vs Linear 26.3%로 강한 이점 유지
